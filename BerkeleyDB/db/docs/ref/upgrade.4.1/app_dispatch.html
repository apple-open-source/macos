Release 4.1: Application-specific logging and recovery

The application-specific logging and recovery tools and interfaces have been reworked in the 4.1 release to make it simpler for applications to use Berkeley DB to support their own logging and recovery of non-Berkeley DB objects. Specifically, the DB_ENV->set_recovery_init and DB_ENV->set_tx_recover interfaces have been removed, replaced by DB_ENV->set_app_dispatch. Applications using either of the removed interfaces should be updated to call DB_ENV->set_app_dispatch. For more information see "Application-specific logging and recovery" and the DB_ENV->set_app_dispatch documentation.
